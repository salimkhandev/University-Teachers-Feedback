                              ABSTRACT
================================================================================
                                                                               v

The University Teachers Feedback System is a comprehensive, AI-powered web 
application designed to streamline the collection, analysis, and visualization of 
student feedback regarding university faculty members. Traditional feedback 
mechanisms in educational institutions often suffer from inefficiencies such as 
paper-based processes, lack of anonymity, delayed results, absence of actionable 
insights, and minimal engagement between teachers and the feedback they receive. 
This system addresses these challenges by providing a digital platform where 
students can submit feedback on their teachers through an intuitive user 
interface, administrators can monitor the entire ecosystem, and teachers can 
receive AI-generated summaries, interactive rating visualizations, and chatbot 
coaching based on student comments.

The system employs a modern technology stack comprising React with Vite for the 
frontend, Node.js with Express for the backend, MongoDB with Mongoose for data 
persistence, and Google Gemini artificial intelligence for natural language 
processing tasks. Role-based access control ensures that students, teachers, and 
administrators each have appropriate views and permissions. The platform 
incorporates intelligent caching strategies to optimize AI API usage, CSV bulk 
import capabilities for rapid data population, and comprehensive analytics 
dashboard with statistical charts. This thesis presents the complete system 
architecture, database schema design, implementation decisions, feature 
specifications, testing strategies, and deployment considerations of this 
production-ready educational technology solution.

1.1 Overview

In the contemporary landscape of higher education, the quality of teaching 
directly influences student learning outcomes, institutional reputation, and 
overall academic excellence. Universities worldwide have recognized the critical 
importance of collecting and analyzing student feedback as a mechanism for 
continuous improvement in teaching quality. However, the process of gathering, 
processing, and deriving actionable insights from student feedback has 
traditionally been fraught with challenges that span administrative, 
pedagogical, and technological domains.

The University Teachers Feedback System is an AI-powered web-based solution 
designed to automate and enrich this process. Utilizing modern web development 
frameworks combined with state-of-the-art Large Language Models (LLMs), the 
system provides a digital platform where students can anonymously and securely 
evaluate their assigned teachers. Simultaneously, it empowers faculty members 
with detailed analytics and AI-generated text summaries of student qualitative 
comments, closing the feedback loop effectively within educational institutions.

1.2 Motivation

Historically, many universities have relied on paper-based feedback forms 
distributed at the end of each academic term. These paper-based systems present 
numerous inefficiencies: they are resource-intensive to print and distribute, 
require manual data entry for analysis, are susceptible to data loss, and 
provide limited anonymity which discourages honest student responses. The 
logistical burden of managing thousands of paper forms across multiple 
departments is substantial, requiring significant staff time. Furthermore, the 
time lag between feedback collection and result dissemination often renders the 
insights less actionable, as the academic term may have already concluded by 
the time teachers receive their evaluations.

Even when institutions have transitioned to basic digital solutions, many 
systems offer little more than simple form submission and quantitative rating 
aggregation. These systems fail to extract meaningful insights from the 
qualitative comments that constitute the richest source of feedback data. The 
inability to automatically analyze text-based feedback represents a significant 
missed opportunity, as student comments contain nuanced observations about 
teaching methodology, communication effectiveness, and classroom management that 
cannot be captured through numerical ratings alone.

The motivation behind this project is to build an intelligent, secure, and 
end-to-end automated platform that leverages Large Language Models—specifically 
Google's Gemini model—to solve both quantitative and qualitative feedback 
bottlenecks. This system transforms raw student comments into coherent, 
unbiased, and actionable development goals for faculty, while providing admins 
with high-level dashboards to monitor institutional performance.

1.3 Problem Statement and Description

Educational institutions face several critical challenges in implementing 
effective teacher feedback systems:

First, there is a total lack of integrated platforms. Institutions often use 
disjointed tools: students submit on one form, admins aggregate results in Excel, 
and teachers receive static PDF reports weeks later. This fragmentation leads to 
data inconsistencies and delays.

Second, feedback analysis is overwhelmingly manual. Processing hundreds or 
thousands of student comments to extract meaningful patterns, identify recurring 
complaints, and summarize strengths/weaknesses requires prohibitive human 
effort. Consequently, qualitative text data is often left unanalyzed.

Third, the feedback loop remains broken. Students do not see any visible impact 
of their feedback, and teachers receive raw feedback dumps without structured, 
constructive guidance, leading to defensive reactions rather than pedagogical 
improvement.

Fourth, system configuration and bulk population are highly complex. Managing 
university organizational structures—comprising departments, semesters, 
sections, subjects, and assignments—manually is prone to error, yet modern 
evaluation tools often lack seamless import capabilities.

The University Teachers Feedback System addresses these problems by providing 
an integrated, role-based platform that automates administrative setup via CSV 
imports, aggregates ratings instantly into dashboards, and applies AI models 
to summarize written comments and act as a conversational teaching coach.

1.4 Proposed Solution

The proposed solution is a modular, responsive Single Page Application (SPA) 
integrated with a secure RESTful API and a Generative AI backend. The solution 
is tailored for three distinct user roles:

- Students: Access a lightweight portal displaying their specific teachers and 
  subjects based on section/semester. They can easily submit ratings (1-10) and 
  written comments, and edit their submissions (with version tracking) as long 
  as the evaluation window remains open.

- Teachers: Access a comprehensive dashboard featuring Recharts-based 
  visualizations of their rating distributions and subject averages. They 
  receive an AI-generated, synthesized summary of student comments, and can 
  interact with a "Chat Coach"—an AI chatbot primed with their actual feedback 
  data that offers personalized, constructive teaching guidance.

- Administrators: Oversee the platform using overview stats, automated ranking 
  charts, student tracking lists, and a multi-step System Setup Wizard. Admins 
  can bulk-import student records via CSV with automated conflict resolution 
  and configure AI API settings, including key rotation and model selection.

1.5 Goal and Objectives

The primary goal of this project is to design, implement, and deploy an 
AI-enhanced student feedback platform that bridges the gap between evaluation, 
analysis, and faculty coaching.

To achieve this, the following objectives have been defined:

1. To implement role-based access control (RBAC) ensuring secure, distinct 
   interfaces and permissions for students, teachers, and administrators.

2. To develop an intuitive feedback submission engine that allows students to 
   rate and comment on teachers, with full update capability and auditing.

3. To integrate Google Gemini AI models for automatic generation of teacher 
   performance summaries from raw student comments.

4. To design an intelligent chatbot coaching interface that allows teachers 
   to converse with their feedback data safely.

5. To provide administrators with granular tracking of student feedback progress, 
   flagging pending submissions across sections and departments.

6. To create a multi-step Setup Wizard including bulk import of student and 
   teacher rosters from CSV files with automatic conflict resolution.

7. To implement intelligent caching for AI-generated summaries to minimize API 
   latency and transaction costs.

8. To deploy the system using a high-availability, modern architecture on cloud 
   platforms (Vercel and MongoDB Atlas).

1.6 Scope of Project

The scope of the project encompasses the development and integration of the 
following modules:

- Domain Configuration: Modeling university departments, semesters, sections, 
  subjects, and teacher-subject-section-semester assignments.

- Authentication Module: JWT-based secure session management featuring 
  access/refresh token rotation, bcrypt password hashing, and login interfaces.

- Student Portal: Progress tracking, teacher card grids, and inline 
  rating sliders with comment validation.

- Teacher Portal: Subject-wise analytics, interactive rating distribution 
  charts, AI summary panel, and the conversational AI Coach.

- Admin Portal: Overview metrics, system setup wizard (Department, Semester, 
  Section, Subject, Teacher, Assignment, Student, and CSV Bulk Import), user 
  accounts CRUD, AI settings key management (rotation, masking, deactivation), 
  and admin-teacher chat report features.

The scope is limited to the web application domain (responsive for mobile and 
desktop). While student identities are tracked internally to prevent duplicate 
submissions, student names and roll numbers are completely stripped from 
feedback records before they are displayed to teachers, maintaining anonymity.

1.7 Limitations of Project

The project is subject to the following technical and operational limitations:

- External API Dependency: The AI-powered summary and chat services rely on 
  the availability of Google's Gemini API. Network latency or upstream service 
  interruptions can affect performance.

- API Quota Limits: Generation of summaries and chat responses is bound by 
  Gemini rate limits (requests per minute). While mitigated by multi-key 
  rotation and caching, extreme usage may result in temporary throttling.

- Internet Connectivity: As a cloud-hosted platform utilizing MongoDB Atlas and 
  external AI services, the system requires active internet connectivity.

- Natural Language Limits: The AI engine parses qualitative data in English. 
  Non-English comments are aggregated but may not be summarized with the same 
  granularity.

2.4 Modern Web Technologies for Education

The selection of appropriate technologies is crucial for the success of any 
educational software system. The University Teachers Feedback System employs a 
modern technology stack chosen for its performance, developer experience, and 
ecosystem support.

React, the frontend library developed by Facebook, has become the dominant 
choice for building user interfaces due to its component-based architecture, 
efficient virtual DOM rendering, and extensive ecosystem. Combined with Vite, a 
next-generation build tool that offers instant server start and fast hot module 
replacement, the frontend delivers a responsive user experience with rapid 
development iteration.

Node.js provides the backend runtime, offering excellent performance for 
I/O-intensive applications typical of web APIs. Express, the most popular 
Node.js web framework, provides a minimalist yet powerful foundation for 
building RESTful APIs with middleware support and routing capabilities.

MongoDB, a NoSQL document database, was selected for its flexible schema design 
that accommodates the hierarchical data structures common in educational 
institutions. Mongoose provides an elegant object modeling interface for MongoDB 
with schema validation, middleware, and query building capabilities.

TailwindCSS offers a utility-first CSS framework that enables rapid interface 
development with consistent design patterns. Combined with React, it enables 
the creation of responsive, accessible, and visually appealing user interfaces.

Google Gemini AI provides the artificial intelligence backbone, offering 
state-of-the-art natural language processing capabilities through a simple API 
interface. The integration of AI directly into the feedback analysis pipeline 
represents a significant advancement over traditional feedback systems.

3.1 Business Requirements

The University Teachers Feedback System must address the core business needs of educational institutions for collecting, analyzing, and acting upon student feedback. The system must provide a digital platform that replaces paper-based feedback processes, enables automated analysis of qualitative comments through AI, and delivers actionable insights to faculty members while maintaining student anonymity. The system must support role-based access for students, teachers, and administrators, and provide comprehensive dashboards for monitoring institutional teaching performance.

3.2 User Requirements

Students require an intuitive interface to view their assigned teachers, submit feedback with ratings and comments, track their completion progress, and edit submissions within the evaluation window. Teachers require detailed analytics showing rating distributions, subject-wise performance metrics, AI-generated summaries of student comments, and an interactive chat interface for personalized coaching. Administrators require system-wide dashboards for monitoring teacher rankings, tracking student feedback completion, managing user accounts, configuring system settings through a setup wizard, and managing AI API keys and model selection.

3.6 Domain Requirements

The system must model the educational domain including departments, semesters, sections, subjects, teacher assignments, and student enrollments. The system must maintain academic year information for roll number generation and support semester-based data partitioning.

3.7 Use-case Descriptions

Use Case 1: Submit Feedback
Actor: Student
Description: A student submits feedback for an assigned teacher including a rating (1-10) and a mandatory comment.
Preconditions: Student is logged in, teacher assignment exists, feedback window is open
Main Flow:
1. Student views their assigned teachers on the dashboard
2. Student selects a teacher to provide feedback
3. Student adjusts rating slider to desired value (1-10)
4. Student enters comment in the text area
5. Student clicks submit button
6. System validates rating and comment are present
7. System saves feedback to database
8. System updates progress indicator
9. System displays success message
Postconditions: Feedback is saved, progress is updated, student cannot submit again for same assignment

Use Case 2: View AI Summary
Actor: Teacher
Description: A teacher views an AI-generated summary of their student feedback.
Preconditions: Teacher is logged in, feedback exists for the teacher
Main Flow:
1. Teacher navigates to AI Summary tab
2. System checks if cached summary exists
3. If cached and current feedback count matches, display cached summary
4. If new feedback exists, show "stale" indicator
5. Teacher clicks "Generate Summary" if stale
6. System collects all feedback data
7. System sends data to Gemini AI with summary prompt
8. System receives and displays generated summary
9. System caches summary with current feedback count
Postconditions: Summary is displayed and cached for future use

                      4. SYSTEM ARCHITECTURE

4.1 Architecture Diagram

      [Figure 4.1: System Architecture Diagram showing three-tier architecture with 
      Frontend (React/Vite), Backend (Node.js/Express), Database (MongoDB Atlas), 
      and External Services (Google Gemini AI). The diagram illustrates the flow 
      of data from user browser through API layer to database and AI services, 
      with security layers including JWT authentication and role-based access 
      control.]

The University Teachers Feedback System follows a modern client-server 
architecture with clear separation between the frontend presentation layer and 
the backend API layer. The architecture is designed for deployment on Vercel's 
serverless platform, which influences several design decisions.

The system architecture consists of three main layers:

1. Frontend Layer: A React single-page application (SPA) built with Vite that 
   runs in the user's browser. The frontend handles all user interface rendering, 
   client-side routing, authentication state management, and API communication 
   through Axios HTTP client with JWT interceptor.

2. Backend API Layer: A Node.js Express application that handles all business 
   logic, data validation, authentication, authorization, and AI integration. 
   The API exposes RESTful endpoints organized by functional domain (auth, 
   setup, student, feedback, teacher, admin).

3. Database Layer: MongoDB Atlas provides the data persistence layer. Mongoose 
   ODM is used for schema definition, validation, query building, and data 
   modeling. The database is organized into 11 collections that model the 
   complete educational domain.

The architecture also integrates with external services:
- Google Gemini AI API for natural language processing
- MongoDB Atlas for database hosting
- Vercel for hosting both frontend and backend

API Layer:

4.4 Database Schema Design

The database schema is designed around 11 MongoDB collections that model the 
complete educational domain:

The Users collection serves as the central identity repository storing 
authentication credentials and user profile information for all three roles 
(admin, teacher, student). The Departments collection contains master data for 
academic departments. The Semesters collection links to departments and 
represents academic terms. The Sections collection links to semesters and 
represents student divisions within a semester.

The Subjects collection links to sections and represents courses taught. The 
TeacherAssignments collection is the critical junction that maps teachers to 
subjects within specific sections and semesters. The Students collection links 
users to their academic context (section and semester) and stores academic 
identifiers.

The Feedback collection stores student ratings and comments linked to specific 
teacher assignments, with a compound unique index on (studentId, assignmentId). 
The AISummaries collection caches AI-generated summaries per teacher. The 
ChatSessions collection stores chat message history per teacher. The 
AISettings collection stores AI configuration including selected model and 
multiple API keys.

4.5 AI Integration Architecture

The AI integration is designed with reliability, cost optimization, and 
scalability in mind. The architecture follows these principles:

Multi-Key Rotation: The system supports configuration of multiple Gemini API 
keys. When a request is made, the system iterates through active keys until one 
succeeds. Failed keys are automatically deactivated if the error indicates an 
invalid key, and success/failure is tracked per key for monitoring.

Caching Strategy: AI summaries are cached in the AISummaries collection. When a 
teacher requests their summary, the system checks the cached version against the 
current feedback count. If the count matches, the cached summary is returned 
immediately. If new feedback exists, a stale indicator is shown and the teacher 
can request regeneration. This approach reduces API costs and improves response 
times.

Rate Limiting: The system implements rate limiting with cooldown periods. When 
a 429 (rate limit) error is received from Gemini, the system calculates the 
retry delay from the error response and prevents further summary generation 
requests for that teacher during the cooldown period.

Context Handling: For chat functionality, the system manages conversation 
history with intelligent context management. When the history exceeds a length 
threshold, the system uses Gemini itself to summarize the previous conversation, 
compacting it into a dense representation. This prevents token limit issues 
while preserving conversation continuity.

Model Selection: Administrators can choose between two Gemini models: 
gemini-2.5-flash for higher quality analysis and gemini-2.5-flash-lite for 
faster, more cost-effective processing. The Lite model is the default for its 
optimal balance of performance and cost.

4.9 API Design and Routes

The API follows RESTful design principles with endpoints organized by functional 
domain. All endpoints except login require JWT authentication. The API base 
path is /api.

Authentication Endpoints:
POST /api/auth/login - Authenticate user with username and password, returns 
access token and refresh token
POST /api/auth/refresh-token - Refresh expired access token using valid refresh 
token

Setup Endpoints (all require admin role):
GET /api/setup/departments - List all departments
GET /api/setup/semesters - List semesters, optionally filtered by department
GET /api/setup/sections - List sections, optionally filtered by semester
GET /api/setup/subjects - List subjects, optionally filtered by section
POST /api/setup/departments - Create a new department
POST /api/setup/semesters - Create semesters for a department
POST /api/setup/sections - Create sections for a semester
POST /api/setup/subjects - Create subjects for a section
POST /api/setup/teachers - Register a new teacher user
POST /api/setup/assignments - Assign a teacher to a subject in a section
POST /api/setup/students - Register a single student
POST /api/setup/students/bulk-csv - Bulk import students via CSV upload

Student Endpoints (require student role):
GET /api/student/me - Get current student's profile
GET /api/student/assignments - Get teacher assignments with feedback status

Feedback Endpoints (require student role):
POST /api/feedback - Submit new feedback
PATCH /api/feedback/:id - Edit existing feedback
GET /api/feedback/mine - Get all feedback submitted by the current student

Teacher Endpoints (require teacher role):
GET /api/teacher/me - Get current teacher's basic info
GET /api/teacher/ratings - Get aggregated rating breakdown per subject
GET /api/teacher/summary - Get AI-generated summary (cached or fresh)
GET /api/teacher/chat - Get chat history
GET /api/teacher/chat/bootstrap - Get chat history and cached summary in one 
request
POST /api/teacher/chat - Send message to AI coach

Admin Endpoints (require admin role):
GET /api/admin/teachers/rankings - Get ranked list of all teachers
GET /api/admin/student-tracking/:sectionId - Track feedback completion by 
section
GET /api/admin/student-tracking/department/:departmentId - Track feedback by 
department with pagination
GET /api/admin/pending-students - Get all students with incomplete feedback
POST /api/admin/report/generate/:teacherId - Generate AI summary for a teacher
GET /api/admin/report/status - Get system statistics
GET /api/admin/ai/settings - Get AI configuration
PATCH /api/admin/ai/settings/model - Update selected AI model
POST /api/admin/ai/settings/keys - Add a new API key
PUT /api/admin/ai/settings/keys/:keyId - Update an existing API key
PATCH /api/admin/ai/settings/keys/:keyId/toggle - Toggle API key active status
DELETE /api/admin/ai/settings/keys/:keyId - Delete an API key
POST /api/admin/report/chat/:teacherId - Chat with AI about a teacher
GET /api/admin/users - Get users list, optionally filtered by role
PUT /api/admin/users/:id - Update user details
PUT /api/admin/users/:id/password - Reset user password
DELETE /api/admin/users/:id - Delete user and all related data

The University Teachers Feedback System is built using a carefully selected 
modern technology stack. Each technology was chosen to address specific

  efficient rendering through virtual DOM, and a mature ecosystem of tools and 
  libraries.

  replacement, and optimized production builds. Vite significantly improves 
  developer experience compared to traditional bundlers like Webpack.
- React Router v6: The standard routing library for React applications, 
  providing declarative routing, nested routes, and route protection through 
  layout routes.
- Recharts: A composable charting library built on React components, used for 
  creating the teacher ranking bar charts and rating distribution visualizations.
- TailwindCSS: A utility-first CSS framework that enables rapid UI development 
  through composable utility classes, ensuring consistent design and responsive 
  layouts.
- Axios: A promise-based HTTP client for making API requests from the browser, 
  providing interceptors for JWT token management and request/response 
  transformation.
- TypeScript: A typed superset of JavaScript that provides static type checking, 
  improving code quality and developer productivity.

Backend Technologies:
- Node.js: The JavaScript runtime providing event-driven, non-blocking I/O 
  model suitable for API servers handling concurrent requests.
- Express: A minimal and flexible Node.js web application framework providing 
  robust routing, middleware support, and HTTP utility methods.
- TypeScript: Providing type safety for backend code, reducing runtime errors 
  and improving code maintainability.
- Mongoose: An Object Data Modeling (ODM) library for MongoDB providing schema 
  validation, query building, middleware (pre/post hooks), population 
  (referencing), and business logic encapsulation.
- bcryptjs: A password hashing library implementing the bcrypt algorithm, 
  providing salt generation and hash comparison for secure password storage.
- jsonwebtoken: A JWT implementation for creating and verifying JSON Web Tokens 
  used for authentication and session management.
- Multer: A middleware for handling multipart/form-data, used for CSV file 
  upload processing.
- csv-parse: A CSV parsing library for processing uploaded CSV files into 
  structured data.
- dotenv: A zero-dependency module for loading environment variables from .env 
  files.

AI Integration:
- @google/generative-ai: The official Google Generative AI SDK for interacting 
  with Gemini models, providing chat session management and content generation.

Database:
- MongoDB Atlas: The fully-managed cloud database service providing automated 
  backups, monitoring, and scaling capabilities.
- MongoDB: The NoSQL document database providing flexible schema design, 
  powerful indexing, and horizontal scaling through sharding.

Deployment:
- Vercel: The cloud platform for static sites and serverless functions, 
  providing continuous deployment from Git, automatic SSL, and global CDN 
  distribution.
